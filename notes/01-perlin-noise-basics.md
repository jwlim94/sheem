## Noise란?

**"좌표를 넣으면 -1~1을 돌려주는, 부드럽지만 불규칙한 함수"**

### Random vs Noise

|                  | Math.random()        | noise(x, z)                        |
| ---------------- | -------------------- | ---------------------------------- |
| 이웃 값과의 관계 | 없음 (독립적)        | 연속적 (비슷한 입력 → 비슷한 출력) |
| 결과             | 지직거림 (TV 노이즈) | 부드러운 곡선 (자연스러운 지형)    |
| 재현 가능?       | 불가                 | seed가 같으면 항상 같은 결과       |

### 핵심 성질

- **연속성**: noise(1.0, 1.0) ≈ noise(1.01, 1.0) → 가까운 입력은 가까운 출력
- **불규칙성**: sin(x)처럼 반복되지 않음 → 내부 랜덤 테이블 때문
- **출력 범위**: -1 ~ 1 (simplex-noise 라이브러리 기준, 실제로는 0 근처에 몰리는 경향)

### Perlin Noise → Simplex Noise 역사

```
1983년: Ken Perlin이 영화 Tron의 CG를 위해 "Perlin Noise" 발명 (아카데미 기술상 수상)
          ↓
        축 방향으로 줄무늬 같은 아티팩트 발견 (정사각형 격자 기반의 한계)
          ↓
2001년: Ken Perlin 본인이 개선판 "Simplex Noise" 발명
```

|                  | Classic Perlin | Simplex                |
| ---------------- | -------------- | ---------------------- |
| 격자 구조        | 정사각형 격자  | 삼각형(simplex) 격자   |
| 축 방향 아티팩트 | 있음           | 없음                   |
| 계산 속도        | 보통           | 더 빠름 (특히 3D 이상) |

"Perlin Noise"는 개념/카테고리 이름이고, 실제로 우리가 쓰는 건 Perlin 본인이 만든 개선판인 **Simplex Noise**.

### 왜 simplex-noise 라이브러리인가

JS에서 쓸 수 있는 noise 라이브러리 중 `simplex-noise`가:

- 가장 가볍고 (0 의존성)
- TypeScript 지원
- 활발히 관리됨
- R3F/Three.js 프로젝트에서 가장 많이 사용

## 지형에서의 사용 원리

```
1. PlaneGeometry 생성 (평평한 격자판, vertex들의 집합)
2. 각 vertex의 (x, z) 좌표를 noise 함수에 입력
3. 리턴된 값을 y 좌표(높이)로 설정
4. → 언덕과 골짜기가 생성됨
```

### PlaneGeometry 구조

```
PlaneGeometry(size, size, segments, segments)
- size: 실제 크기 (미터). 원점 중심이므로 범위는 -size/2 ~ +size/2
- segments: 칸 수. vertex 수 = (segments+1)^2

예) PlaneGeometry(500, 500, 200, 200)
    - x, z 범위: -250 ~ 250
    - vertex 수: 201 x 201 = 40,401개
    - XY 평면으로 생성되므로 rotateX(-PI/2)로 XZ 평면으로 회전 필요
```

## 주요 파라미터

### scale (noise 입력 스케일 = frequency)

noise 함수의 "얼마나 넓은 영역을 보느냐"를 결정.

```
vertex.y = noise(x * scale, z * scale) * height
```

- **작은 scale (0.005)**: 이웃 vertex 간 입력 차이가 작음 → 완만한 언덕 (SlowRoads 느낌)
- **큰 scale (0.05)**: 이웃 vertex 간 입력 차이가 큼 → 울퉁불퉁

핵심: scale은 출력 범위(-1~1)를 바꾸지 않음. **변화의 빈도(frequency)**를 바꿈.

**주의: vertex의 물리적 거리가 변하는 게 아님.**
scale이 바뀌어도 vertex들의 x, z 위치는 그대로. 바뀌는 건 **noise 함수에 들어가는 입력값의 차이**.
noise한테 "너희 더 멀리 있어"라고 속이는 것과 같음.

```
vertex A: x=0, vertex B: x=2.5  (물리적 거리: 항상 2.5)

scale 0.008 → noise 입력: 0.000 vs 0.020  (차이 0.020) → 비슷한 y → 완만
scale 0.05  → noise 입력: 0.000 vs 0.125  (차이 0.125) → 다른 y  → 가파름
```

### height (높이 배율 = amplitude)

noise 출력(-1~1)에 곱해서 실제 높이로 변환.

```
height = 8 → 지형 높이 범위: 약 -8m ~ +8m (이론적, 단일 octave)
```

## FBM (Fractal Brownian Motion)

**단일 noise의 한계**: noise는 불규칙하지만, **한 가지 크기의 불규칙**만 있음.

```
noise(x * 0.008)로 만든 지형:

   ∿      ∿       ∿        ∿      ∿

언덕이 어디에 생기는지는 불규칙. 높이도 제각각.
하지만 언덕의 "폭"은 전부 비슷함. 전부 다 ~100m급 언덕.
작은 2m짜리 울퉁불퉁은 없고, 거대한 1km짜리 산맥도 없음.
→ 오로지 한 사이즈의 언덕만 반복되는 지형.
```

현실 지형은 큰 산맥(수km) 위에 언덕(수백m)이 있고, 그 위에 바위 굴곡(수m)이 있음.
**크기가 다른 패턴이 동시에 공존**하는 구조. noise 하나는 이 중 딱 한 층만 만들 수 있음.

### 원리

여러 겹(octave)의 noise를 합성. 각 겹마다:

- **주파수(frequency) × 2**: 더 세밀한 패턴
- **높이(amplitude) × persistence**: 영향도를 줄임

```
octave 0: freq=0.005, amp=8.0    → 넓고 큰 언덕 (±8m)
octave 1: freq=0.010, amp=2.8    → 중간 굴곡 (±2.8m)
octave 2: freq=0.020, amp=0.98   → 작은 울퉁 (±0.98m)
octave 3: freq=0.040, amp=0.34   → 미세 디테일 (±0.34m)
```

### persistence (거칠기 조절)

각 octave의 amplitude를 줄이는 비율. **FBM의 핵심 파라미터**.

- **0.5 (표준)**: 각 겹이 절반씩 영향 → 적당한 디테일
- **0.35 (부드러움)**: 작은 굴곡 영향 적음 → 넓고 매끄러운 초원 (SlowRoads)
- **0.3 (매우 부드러움)**: 거의 큰 언덕만 보임
- **0.6 (거칠음)**: 작은 디테일이 많음 → 바위산 느낌

실제 amp 감쇠 비교 (height=10 기준):

```
persistence = 0.35 (우리 설정, 부드러움):
  octave 0: amp = 8     (100%)  ← 거의 이것만 보임
  octave 1: amp = 2.8   (35%)
  octave 2: amp = 0.98  (12%)
  octave 3: amp = 0.34  (4%)   ← 거의 영향 없음

persistence = 0.6 (거친 바위산):
  octave 0: amp = 10    (100%)
  octave 1: amp = 6     (60%)
  octave 2: amp = 3.6   (36%)  ← 여전히 큰 영향
  octave 3: amp = 2.16  (22%)  ← 작은 디테일이 강하게 남음
```

### 구현 코드

```typescript
const octaves = 4;
let y = 0;
let freq = scale; // 시작 주파수 (0.005)
let amp = height; // 시작 높이 (8)

for (let o = 0; o < octaves; o++) {
  y += noise2D(x * freq, z * freq) * amp;
  freq *= 2; // 주파수 2배 (더 세밀한 패턴)
  amp *= persistence; // persistence로 감쇠 (0.4면 40%씩 줄어듦)
}
```

### 왜 더하면 디테일이 살아나는가

핵심: **freq가 다르면 각 octave가 "다른 크기의 패턴"을 만든다.**

같은 freq로 4번 더하면 그냥 amp 4배인 것과 같아서 의미 없음.
freq가 매번 2배가 되니까 각 octave가 **다른 스케일의 디테일**을 담당하게 됨.

#### vertex 5개 시뮬레이션 (x = 0, 2.5, 5, 7.5, 10)

**octave 0** (freq=0.008): noise 입력 차이가 0.02씩 → **거의 비슷한 값** → 완만한 경사
**octave 1** (freq=0.016): noise 입력 차이가 0.04씩 → 변화가 좀 더 있음
**octave 2** (freq=0.032): noise 입력 차이가 0.08씩 → 들쭉날쭉
**octave 3** (freq=0.064): noise 입력 차이가 0.16씩 → 가장 들쭉날쭉

```
vertex:       [0]       [2.5]      [5]       [7.5]      [10]

octave 0:     0.0        0.2       0.4        0.5        0.6    ← 완만한 경사
octave 1:     0.0        0.2       0.32      -0.12       0.4    ← 중간 굴곡
octave 2:     0.3       -0.5       0.8       -0.2        0.1    ← 작은 울퉁
octave 3:    -0.1        0.4      -0.3        0.5       -0.2    ← 미세 디테일
─────────────────────────────────────────────────────────────
합계 y:       0.2        0.3       1.22       0.68       0.9
```

- **octave 0만**: `0.0 → 0.2 → 0.4 → 0.5 → 0.6` → 부드러운 경사 하나. 끝.
- **4개 합산**: `0.2 → 0.3 → 1.22 → 0.68 → 0.9` → 전체적으로 오르막(octave 0)이지만 중간중간 올록볼록(octave 1-2), 미세하게 울퉁불퉁(octave 3)

→ **큰 곡선의 "형태" 위에 작은 곡선이 "얹히는" 구조.**

#### 바다 파도 비유

```
큰 파도 (swell)        :  ~~~~~~~~~~~~~~~     ← octave 0 (느린 주기, 큰 진폭)
중간 파도 (wave)       :  ~~  ~~  ~~  ~~      ← octave 1
잔물결 (ripple)        :  ~ ~ ~ ~ ~ ~ ~ ~     ← octave 2
미세한 떨림 (shimmer)  :  ~~~~~~~~~~~~~~~~     ← octave 3

합치면 → 현실의 바다 수면
```

큰 파도 하나만 있으면 사인파처럼 단조로운데,
크기가 다른 파도를 **그냥 더하기만 해도** 현실적인 수면이 됨. 지형도 같은 원리.

#### 더해서 숫자 하나가 되는데 왜 패턴이 보이는가?

vertex 하나만 보면 그냥 숫자 하나 (예: 3.7). 거기서 "4가지 패턴"은 안 보임.
하지만 **모든 vertex가 같은 공식을 거치기 때문에**, vertex들을 쭉 나열하면 패턴이 드러남.

지형도 동일. 각 vertex는 숫자 하나지만, **공간에 걸쳐 보면** 4가지 크기의 패턴이 모두 보임.

#### vertex 수는 변하지 않음

FBM은 vertex를 추가하는 게 아님. vertex 수는 PlaneGeometry가 정한 40,401개 그대로.
FBM은 각 vertex의 **y값을 계산하는 과정**일 뿐:

```
vertex #7324 (x=12.5, z=-30.0):

y = 0
y += noise(... × 0.008) × 10    = +3.0   ← octave 0
y += noise(... × 0.016) × 4     = -1.5   ← octave 1 더함
y += noise(... × 0.032) × 1.6   = +0.8   ← octave 2 더함
y += noise(... × 0.064) × 0.64  = -0.2   ← octave 3 더함
────────────────────────────────────────
y = 2.1  ← 이 vertex의 최종 높이 (하나의 숫자)
```

### scale ↑ vs height ↑ vs persistence ↑ 차이

셋 다 "위아래로 커진다"는 결과는 비슷하지만 성격이 다름:

- **scale ↑** → 같은 거리 안에 **언덕 수가 늘어남** (울퉁불퉁). 전체 지형이 촘촘해짐.
- **height ↑** → 언덕 수는 같은데 **각 언덕이 더 높아짐**. 전체가 스케일업.
- **persistence ↑** → 큰 언덕 형태는 유지되는데 **작은 굴곡이 강해져서 표면이 거칠어짐**.

persistence를 극단적으로 올려보면 (0.7+) scale/height와 느낌이 다름.
큰 형태는 그대로인데 디테일만 거칠어지는 것을 확인할 수 있음.

### lacunarity (주파수 배수) = 국룰 2.0

각 octave마다 freq에 곱하는 값을 **lacunarity**라고 부름.

```typescript
freq *= 2; // lacunarity = 2.0
```

거의 모든 구현에서 2.0을 사용. 이유:

- 음악의 옥타브처럼 주파수 2배가 자연스러운 프랙탈 스케일링
- 실제 자연 지형의 프랙탈 분석 결과와도 일치
- 다른 값(1.8, 2.2 등)을 써도 눈에 띄는 차이가 거의 없음

→ **보통 건드리지 않음. 2배 고정.**

### octaves 수 가이드

| octaves | 용도                              |
| ------- | --------------------------------- |
| 2-3     | 모바일, 성능 우선. 디테일 적음    |
| **4**   | **가장 일반적. 성능/품질 균형**   |
| 5-6     | 고품질 지형. 클로즈업이 많은 경우 |
| 8+      | 오프라인 렌더링, 영화급           |

4를 넘어가면 추가되는 디테일이 너무 작아서 (amp가 이미 거의 0) 눈에 안 보이는데
계산 비용만 늘어남. 그래서 4가 실시간 렌더링의 표준.

### FBM 파라미터 조절 우선순위

| 파라미터        | 보통 조절하나? | 이유                        |
| --------------- | -------------- | --------------------------- |
| **persistence** | **항상 조절**  | 지형 스타일을 결정하는 핵심 |
| **octaves**     | 가끔 조절      | 성능 vs 디테일 트레이드오프 |
| **lacunarity**  | 거의 안 건드림 | 2.0이 국룰                  |

### SlowRoads와의 관계

SlowRoads도 Perlin noise 변형 + 여러 octave 합성으로 지형 생성.
"Fractal noise" 기법은 거의 모든 procedural 지형 생성에 사용됨.

## Seed와 Alea

### 문제: JavaScript의 Math.random()은 seed를 줄 수 없음

```javascript
Math.random() → 매번 다른 값, 재현 불가
```

### 해결: Alea (시드 기반 PRNG)

```javascript
const rng = Alea("my-seed")
rng() → 0.4821  // 같은 seed면 항상 같은 값
```

### 지형에서의 의미

```
같은 seed → 같은 랜덤 테이블 → 같은 noise 결과 → 같은 지형
다른 seed → 다른 랜덤 테이블 → 다른 noise 결과 → 다른 지형
```

SlowRoads: 유저가 seed를 공유하면 같은 맵을 경험할 수 있음.
Sheem: 멀티플레이어이므로 고정 seed로 모든 유저에게 같은 맵 제공 예정.

## SlowRoads에서 배운 기법 (참고)

- Perlin noise 변형 + Alea PRNG로 무한 지형 생성
- FBM으로 자연스러운 다단계 지형 (큰 언덕 + 작은 굴곡)
- 단일 Ground 셰이더: 경사도 기반으로 풀/절벽/모래 텍스처 자동 블렌딩
- LOD: 도로 주변은 고해상도, 먼 곳은 저해상도
- Instanced geometry로 풀/나무 대량 렌더링
- 오브젝트 풀링 + 메모리 사전 할당으로 GC 방지
