# Sheem (쉼) — Project Plan

> "쉼" — 한국어로 휴식, 잠시 멈춤. 자연의 소리와 3D 공간이 만나는 힐링 소셜 플랫폼.

---

## 1. 프로젝트 개요

### 컨셉
고퀄리티 ASMR 자연음을 3D 공간에서 경험하는 멀티유저 웹 플랫폼. 유저는 3D 맵을 돌아다니며 위치에 따라 달라지는 소리를 경험하고, 같은 소리를 좋아하는 사람들과 자연스럽게 모여 채팅하거나 조용히 공존할 수 있다.

### 핵심 가치
- **공간적 오디오**: 같은 빗소리라도 어디서 듣느냐에 따라 다르게 들린다 (텐트 안, 숲속, 차 안, 처마 밑, 아스팔트 위)
- **소셜 레이어**: 같은 소리를 좋아하는 사람들이 자연스럽게 같은 공간에 모임
- **감성 기반 집중 공간**: 공부, 작업, 휴식을 위한 ambient 환경

### 브랜드명
- **영문**: Sheem
- **어원**: 한국어 "쉼" (휴식, 멈춤)
- **발음**: 영어권에서 자연스럽게 읽히면서도 부드럽고 조용한 어감. "Shh-"로 시작하는 소리 자체가 앱 컨셉과 일치

## 2. 기술 스택

### 렌더링 / 3D
- **Three.js** — 웹 3D 엔진 (환경, 지형, 셰이더, 카메라)
- **WebGL Shaders (GLSL)** — 날씨, 물, 안개, 빛 효과

### 오디오
- **Web Audio API** — 오디오 처리, 믹싱, 크로스페이드
- **Three.js PositionalAudio** — 3D 공간 기반 소리 (거리에 따른 볼륨 자동 계산)

### 멀티유저
- **Socket.io** — 실시간 유저 위치 동기화, 채팅

### 에셋
- **Three.js (코드 생성)**: 지형, 하늘, 날씨, 안개, 물, 빛, 그림자
- **Blender**: 맵 위 소품 (텐트, 벤치, 버섯 등 단순 오브젝트부터 시작)
- **구매 에셋**: 캐릭터 (초기에는 `.blend` 소스 포함 에셋 구매, 추후 커스터마이징)

## 3. 오디오 시스템 설계

### 두 가지 레이어 조합

```
Zone 기반 (배경음)         +    PositionalAudio (오브젝트 기반)
─────────────────────────────────────────────────────────────
숲 구역 배경 빗소리 (낮게)       나뭇잎 위 빗소리 (나무 위치)
                                 웅덩이 빗소리 (웅덩이 위치)
텐트 구역 배경음 (실내감)        텐트 천막 소리 (텐트 위치)
```

- **Zone 전환**: 구역 경계에서 두 소리가 자연스럽게 크로스페이드
- **Positional**: 오브젝트 가까이 갈수록 해당 소리 커짐 (Three.js 자동 처리)

### 오디오 소싱
- Freesound.org, Soundsnap, Epidemic Sound 등에서 고퀄리티 자연음 수집
- 공간감 있는 binaural / spatial audio 우선 선별

## 5. 캐릭터 커스터마이징 시스템

### 작동 방식
미리 조합을 렌더해두는 게 아니라 **런타임 레이어드 시스템**:

```
베이스 캐릭터 (skeleton)
    ├── 머리카락 메시 (같은 skeleton 공유)
    ├── 상의 메시 (같은 skeleton 공유)
    ├── 하의 메시 (같은 skeleton 공유)
    └── 소품 메시 (본에 attach)
```

- 모든 의상은 베이스 캐릭터와 **동일한 리그 구조**로 제작
- 유저 구매 아이템은 DB에 저장 → 로그인 시 불러와서 장착

### 에셋 구매 시 필수 확인
- `.blend` 소스 파일 포함 여부
- 상업적 사용 허가
- 수정 허가
- 파생 상품 판매 허가 (의상 아이템 판매 시)

> ⚠️ 베이스 캐릭터가 확정되면 모든 의상이 그 리그 기준으로 만들어지므로, **베이스 캐릭터 선택이 가장 중요한 결정**

## 10. Repository 정리 계획

### 현황 파악 (Claude Code로 sheem repo 열면 먼저 확인할 것)

Claude Code를 sheem repo에서 열고 아래를 물어볼 것:

```
지금 이 repo의 상태를 파악해줘:
1. 어떤 스택으로 세팅되어 있어? (package.json, 주요 deps 확인)
2. 현재 main 브랜치에 뭐가 있어?
3. Vercel 연결 설정이 있어? (vercel.json 또는 .vercel 폴더)
4. 지금 돌아가는 게 있으면 어떤 URL이야?
5. 이전 작업물을 legacy 브랜치로 옮기고 main을 새로 시작하려는데,
   어떻게 하면 좋을지 전략 제안해줘.
```

### 목표 구조

```
main 브랜치          → 메인 URL (새 Sheem 작업)
legacy 브랜치        → 별도 URL 또는 보관용 (이전 작업)
```

### 예상 분기 전략 (스택에 따라 달라질 수 있음)

**기존 스택이 새 계획과 겹친다면** (예: 이미 Next.js + Three.js):
- 기존 코드를 `legacy` 브랜치로 push
- main을 클린하게 새 시작

**기존 스택이 완전히 다르다면** (예: 순수 Vite + Three.js):
- 기존 코드를 `legacy` 브랜치로 보존
- main에 Next.js 기반으로 새로 init

**Vercel 분기**:
- `legacy` 브랜치 → Vercel Preview URL로 접근 가능 (자동)
- `main` 브랜치 → 메인 도메인

## 11. Three.js Journey 레퍼런스 활용법

Three.js Journey 전 챕터를 마치며 각 레슨의 핵심 개념과 코드를 md 파일로 정리해두었다. 구현 중 막히는 부분이 생기면 해당 파일을 먼저 확인하고, 개념을 이해한 뒤 구현한다.

### 파일 위치

```
docs/threejs-journey/
├── basics.md              # Ch01 (01~13): Transform, 지오메트리, 텍스처, 머티리얼
├── classic-techniques.md  # Ch02 (14~19): 조명, 그림자, 파티클, 갤럭시
├── advanced-techniques.md # Ch03 (20~26): 물리, 모델 임포트, 레이캐스터, 환경맵
├── shaders.md             # Ch04 (27~44): GLSL 셰이더 전체 (핵심)
├── extra.md               # Ch05 (45~48): 포스트 프로세싱, 성능, 로딩, HTML 혼합
├── portal-scene.md        # Ch06 (49~52): Blender 베이킹 파이프라인 + Three.js 구현
└── r3f.md                 # Ch07 (53~67): React Three Fiber, Zustand, Rapier
```
